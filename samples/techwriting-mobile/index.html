Marauder's Map Annotated iPad Wireframes

Art, copy and layout by Louis Rawlins

Overview

This app provides an island of sanity for staff in the Emergency Department. With the app, they can locate one another, find patients and find equipment to make their day run smoothly and provide the best care for patients.

Wireframes are meant as a conversation between the people that make a product and those that use the product. In this case, the designers, developers and managers as well as the nurses, doctors and staff in the Emergency Department.

When reviewing these wireframes, know there may be inaccuracies and take that as an opportunity to start a dialogue about what you see. This document is meant as a guide.

How to Use This Document

Below are two modes of interpreting the wireframes you see on the following pages.

Interactions

This section shows the actions a user can take with the app. It also shows what information may display from a back-end service.

Calls Outs

This section shows any areas of interest like user interface elements that need modification, or known discrepancies between the wireframes, prototype and the visual design.

Screen 2: Sign In

Clinical sign in screen for staff using National User ID (NUID) and password.

Interactions

  1. Enter NUID with keyboard
  2. Enter password with keyboard
  3. Sign in button to submit credentials; brings user to main screen
  4. Sign in help; brings user to Sign In Help
  5. Terms and Conditions; brings user to Terms and Conditions
  6. Privacy Statement; brings user to Privacy Statement

Calls Outs

  1. Fade animation from Sign In to main screen

Screen 4: Terms and Conditions

Legal agreement prior to use of the app. Can be reviewed from Settings screen.

Interactions

  1. Depending on entry point, sends user back to Sign In or Pod View, Settings
  2. Sends email of Terms and Conditions to user based on device-populated email (and free text entry)
  3. On first use of app, if declined, sends user back to Sign In
  4. On first use of app, if accepted, sends user to main screen

Calls Outs

None

Screen 6: Pod View, Staff List

A view of the Emergency Department, restricted to a single pod, showing waiting area and a list of staff within that pod.

Interactions

  1. Sends user back to Emergency Department view
  2. Sends user to Pod View, Settings
  3. As-you-type results for search, filtered by Staff, Patients, or Equipment
  4. Filter for list view
  5. List view for all available staff as restricted by 2 in Pod A; tapping a list entry displays a detail on the left-hand panel
  6. Double-tapping in a non-active portion of the menu bar locks screen to display as Pod View, Locked

Calls Outs

  1. User signed in to app; Faye Davis (nurse)
  2. Current location in map view; Pod A
  3. Dirty room to let nurses know which rooms need to be cleaned
  4. Patient in room; shows room number, patient name, sex, age, chief complaint; left-hand side shows status color along with time bar in 3 hour segments, filling from bottom-to-top for 12 total hours
  5. Clean room to let nurses know which rooms are ready for new patients
  6. Waiting area for patients that have been triaged, but not in a room

Screen 7: Pod View, Locked

Interactions

  4. Enter password with keyboard

Calls Outs

Screen 13: Pod View, Settings

Settings screen for user help and advanced functionality like "Ready" signal.

Interactions

  1. Sends user back to Emergency Department view
  2. Sends user to Pod View, Settings
  3. Send user to main screen, like Pod View, Patient Room (in this case, back to the equipment list)
  4. Sends user to Support screen, like Sign In Help
  5. Sends user to App Info screen
  6. Double-tapping in a non-active portion of the menu bar locks screen to display as Pod View, Locked
  7. Signs out user from the app
  8. Signals "Ready" status
